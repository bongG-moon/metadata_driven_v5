# WB 당일 생산량과 공정 그룹 정의 조회

## 문서 정보

- Workflow key: `wb_daily_production_metadata`
- 목적: 당일 WB 공정 생산량을 조회한 뒤, 같은 답변에 등록된 WB 공정 그룹 정의를 함께 제공한다.
- 실행 단계: 2개
- 데이터 handoff: 사용하지 않음
- 기본 오류 정책: 첫 단계 실패 시 중단

## 업무 요청 예시

```text
오늘 WB공정 생산량을 알려주고, 등록된 WB 공정 그룹 정의도 함께 알려줘.
```

## 단계 정의

### 1. 당일 생산량 조회

- `step_id`: `production`
- `tool_name`: `run_data_analysis`
- 질문: `오늘 WB 공정 생산량을 조회해.`
- 선행 단계: 없음
- handoff: `none`
- 실패 정책: `stop`

날짜는 Data Analysis가 실행 시점의 오늘 날짜를 각 조회 작업의 필수 파라미터로 해석한다. 이 단계가 실패하면 실제 수치 없이 정의만 보여주는 불완전한 답변을 피하기 위해 전체 Workflow를 중단한다.

### 2. WB 공정 그룹 정의 조회

- `step_id`: `metadata`
- `tool_name`: `run_metadata_qa`
- 질문: `메타데이터에 등록된 WB 공정 그룹 정의와 포함 공정을 알려줘.`
- 선행 단계: `production`
- handoff: `none`
- 실패 정책: `continue`

2단계는 1단계가 끝난 뒤 실행해야 하지만 생산량 결과 행을 입력으로 사용하지 않는다. 따라서 `result_ref`를 넘기지 않는다.

## 기대 답변

최종 답변은 단계별 완료 문구가 아니라 실제 하위 Flow의 요약을 사용해 다음 두 내용을 함께 보여준다.

1. 기준일과 WB 공정 생산량
2. 등록된 WB 별칭과 포함 공정 목록

한 단계가 실패하면 성공한 결과와 실패 원인을 분리해서 표시한다.
