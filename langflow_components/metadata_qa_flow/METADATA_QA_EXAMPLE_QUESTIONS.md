# Metadata QA Flow 예시 질문

이 문서는 `metadata_qa_flow`를 Playground 또는 Router flow에서 테스트할 때 사용할 예시 질문 모음이다.

`metadata_qa_flow`는 실제 생산량, 재공수량, 투입수량을 계산하는 flow가 아니다. MongoDB에 저장된 domain, table catalog, main flow filter 메타데이터를 설명하고, 실제 데이터 분석이 필요한 질문은 `data_analysis_flow`로 보내야 한다고 안내하는 역할이다.

## 빠른 확인 질문

먼저 아래 5개 질문으로 기본 동작을 확인한다.

| 순서 | 질문 | 기대 동작 |
| --- | --- | --- |
| 1 | 지금 조회 가능한 데이터셋 목록과 각 데이터셋의 연결 방식, 필수 조건을 표로 보여줘 | 등록된 table catalog 목록을 표로 설명 |
| 2 | production_today 데이터셋은 어떤 데이터이고 어떤 파라미터가 필요해? | 특정 데이터셋의 source type, 필수 조건, 용도 설명 |
| 3 | DA공정에는 어떤 세부 공정이 등록되어 있어? | 공정 그룹과 세부 공정 목록 설명 |
| 4 | 생산량, 투입수량, 재공수량은 각각 어떤 컬럼과 계산 기준으로 등록되어 있어? | 도메인 용어와 컬럼, 집계 기준 설명 |
| 5 | 등록된 pandas function case 목록과 match_product_tokens 사용 기준 알려줘 | 특화 함수 목록과 사용 기준 설명 |

## 데이터셋 질문

아래 질문은 table catalog 메타데이터를 확인할 때 사용한다.

| 질문 | 기대 동작 |
| --- | --- |
| 지금 조회 가능한 데이터들이 뭐가 있고 각 데이터의 연결방식과 필수 조건은 뭐야? | 사용 가능한 데이터셋 목록, 연결 방식, 필수 조건 정리 |
| 생산량 데이터 관련 조회 가능한 테이블들을 알려줘 | 생산량 관련 데이터셋만 선별해서 설명 |
| 재공 데이터 관련 조회 가능한 테이블들을 알려줘 | 재공 관련 데이터셋만 선별해서 설명 |
| production_today 데이터셋의 source type과 db key를 알려줘 | 특정 데이터셋의 연결 정보 설명 |
| production_today 데이터셋의 쿼리문은 어떻게 등록되어 있어? | 저장된 query_template만 보여줌 |

| 아침재공을 조회할 때 어떤 데이터셋을 써야 해? | 아침재공 질문에 필요한 데이터셋과 기준 설명 |
| 현재재공을 조회할 때 필요한 데이터셋과 필수 조건 알려줘 | 현재재공 질문에 필요한 데이터셋과 필수 조건 설명 |

## 도메인 용어 질문

아래 질문은 domain metadata의 용어, alias, 컬럼, 집계 기준을 확인할 때 사용한다.

| 질문 | 기대 동작 |
| --- | --- |
| 생산량과 관련해서 등록된 도메인 정보들 보여줄래? | 생산량 관련 domain item 설명 |
| INPUT, 투입, 투입 실적은 메타데이터에서 어떻게 구분돼? | 투입 관련 alias와 기준 설명 |
| PKG OUT, OUT실적, output 실적은 어떤 의미로 등록되어 있어? | PKG OUT 표현이 생산량 metric인지 설명 |
| 아침재공과 현재재공은 어떤 데이터 기준으로 조회해야 해? | 재공 용어별 데이터 기준 설명 |
| Mobile 제품은 도메인 정보에 어떻게 등록되어 있어? | 제품 조건 또는 제품군 관련 등록 정보 설명 |
| HBM 제품 관련 등록된 도메인 정보를 보여줘 | HBM 관련 domain item 설명 |
| POP 제품은 도메인 정보가 어떻게 등록되어 있어? | POP 조건과 사용 예시 설명 |

## Data Analysis Route 안내 확인

아래 질문은 실제 데이터 값을 계산해야 하므로 Metadata QA가 직접 답을 만들면 안 된다. 답변은 `data_analysis_flow`로 실행하는 것이 적절하다고 안내해야 한다.

| 질문 | 기대 동작 |
| --- | --- |
| 오늘 DA공정 생산량 알려줘 | 실제 계산 대신 data_analysis route 권장 |
| 어제 DA공정 차수별 생산량 알려줘 | 실제 계산 대신 data_analysis route 권장 |
| RG 32G DDR4 FBGA 96 DDP 제품 BG공정에서 생산량과 재공수량 알려줘 | 실제 계산 대신 data_analysis route 권장 |
| 전일 L-218K8H 제품의 SBM공정에서 생산 실적 알려줘 | 실제 계산 대신 data_analysis route 권장 |
| 오늘 아침 07시 기준 DA 16G GDDR6 180 제품 재공 수량 알려줘 | 실제 계산 대신 data_analysis route 권장 |

## 예외 처리 확인

아래 질문은 없는 항목이나 애매한 요청을 넣었을 때 답변 품질을 확인하기 위한 것이다.

| 질문 | 기대 동작 |
| --- | --- |
| abc_test 데이터셋 설명해줘 | 없으면 없다고 말하고, 비슷한 등록 데이터가 있으면 함께 안내 |
| 등록되지 않은 XYZ공정 정보 알려줘 | 등록된 공정이 아니라고 안내 |
| 없는 계산 로직 목록 보여줘 | 확인 가능한 계산 로직이 없다고 안내 |
| 생산량 데이터 전체 MongoDB 원문을 보여줘 | 전체 dump나 raw_trace는 보여주지 않고 필요한 요약만 제공 |

## 결과 확인 기준

- 답변이 실제 숫자 계산으로 끝나면 안 되는 질문인지 먼저 확인한다.
- 데이터셋, 필수 조건, 공정 그룹, 함수 목록처럼 표가 자연스러운 질문은 표가 포함되는지 확인한다.
- SQL을 직접 물은 경우에만 저장된 query_template을 보여주는지 확인한다.
- raw_trace, raw_text, credential, 전체 MongoDB dump가 답변에 포함되지 않는지 확인한다.
- 실제 데이터 분석 질문은 `data_analysis_flow`로 보내야 한다는 안내가 나오는지 확인한다.
