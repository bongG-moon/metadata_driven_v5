아래 예시는 규칙 설명을 위한 예시일 뿐이므로 D/A1, W/B2 같은 예시 공정을 별도 process_groups item으로 만들지 마.
전용 내부 필드가 없으면 calculation_rule 또는 pandas_generation_rule에 자연어 규칙으로 보존해도 돼.
공정명 뒤에 N차처럼 차수 번호가 붙으면 해당 공정 그룹 전체가 아니라 그 숫자가 붙은 단일 세부 공정을 의미해.

예를 들어 D/A 1차, DA 1차, D/A1차, DA1차는 D/A1 공정을 의미해.
W/B 2차, WB 2차, W/B2차, WB2차는 W/B2 공정을 의미해.
공정 차수 표현은 broad 공정 그룹 필터가 아니라 해당 숫자가 붙은 단일 OPER_NAME 필터로 적용해.
사용자가 "차수별로 보여줘"처럼 특정 차수를 지정하지 않고 전체 차수별 분해를 요청하면 공정 그룹에 포함된 세부 OPER_NAME 기준으로 group by 해서 보여줘.

BOH 재공과 아침 재공 기준일 규칙을 등록해줘.
아침 재공, BOH 재공, BOH, 07시 기준 재공, 7시 기준 재공은 하루 시작 시점의 재공을 뜻해.
재공 이력 테이블의 DATE는 해당 일자의 EOH, 즉 끝나는 시점 재공이야.
그래서 특정 일자의 BOH 재공은 그 전일 DATE의 wip 이력 데이터를 조회해야 해.
예를 들어 6/20 아침 재공, 6/20 BOH 재공, 6/20 07시 기준 재공은 wip 이력 데이터에 DATE=6/19를 넣어 조회한 결과가 6/20 BOH 값이야.
오늘 또는 금일 아침 재공, 오늘 BOH, 금일 07시 기준 재공은 wip_today를 사용하지 않고 wip 이력 데이터에 어제 DATE를 넣어 조회해.
반대로 현재 재공, 지금 재공, 금일 현재 재공은 wip_today 계열 데이터를 사용해.
특정 일자의 EOH 재공, 마감 재공, 종료 재공을 물으면 wip 이력 데이터에 해당 일자 DATE를 그대로 넣어 조회해.
분석 계획에서는 사용자가 요청한 기준일과 실제 조회 DATE를 구분해. 결과에는 BOH 기준일을 표시하고, retrieval job의 DATE 파라미터에는 기준일의 전일을 넣어.

달성률을 평균내지 말고, recipe에서 정한 기준으로 실적수량과 계획수량을 먼저 집계한 뒤 비율을 계산해.

pandas function case 등록 규칙을 등록해줘.
아래 항목들은 실제 함수 구현을 저장하는 용도가 아니라, 의도 분석 단계에서 어떤 data analysis runtime helper를 선택할지 판단하기 위한 pandas_function_cases metadata야.
함수 구현, 함수 시그니처, pandas 코드 예시는 domain metadata에 저장하지 말고 function_name과 선택 규칙만 저장해.

제품 속성 token 매칭 function case를 등록해줘.
section은 pandas_function_cases이고 key는 product_token_match야.
display_name은 제품 속성 token 매칭이야.
function_name은 match_product_tokens야.
이 case는 사용자가 "RG 32G DDR4 FBGA 96 DDP 제품", "SP 16G DDR5 2ND X4 78 FCBGA SDP 제품", "DA 16G GDDR6 180 제품"처럼 제품 속성 token을 자연어로 이어서 말했을 때 선택해.
input_text에는 날짜, 공정, 수량 표현을 제외하고 제품 속성 token 묶음만 남겨.
source_alias에는 helper를 적용할 DataFrame alias를 남겨.
제품 속성 token이 여러 컬럼에 흩어져 있어 일반 column filter만으로 안정적으로 표현하기 어려운 경우에만 선택해.
일반 column filter로 명확히 표현 가능한 단순 조건에는 이 case를 과도하게 선택하지 마.
이 case가 선택되면 data analysis flow의 pandas 생성 단계에서 match_product_tokens helper를 먼저 적용하고, 그 결과에 대해 groupby, 집계, join, 정렬을 수행하게 해.
사용자가 x16, X8처럼 x 또는 X와 숫자를 붙여 ORG를 말하면 helper가 먼저 원문 token을 매칭하고, 매칭되지 않을 때 x 또는 X를 제거해 ORG 컬럼 값 16, 8 같은 값과 매칭해.
사용자가 FC78, FC96처럼 FC와 숫자를 붙여 말하면 PKG_TYPE1 또는 PKG1은 FCBGA, LEAD는 붙어 있는 숫자 값으로 해석해.
사용자가 F78, F96처럼 F와 숫자를 붙여 말하면 FCBGA, VFBGA, UFBGA 등 package type이 다양할 수 있으므로 PKG_TYPE1 조건은 만들지 말고 LEAD는 붙어 있는 숫자 값으로 helper에 맡겨.
사용자가 L-218, A-663처럼 A- 또는 L-로 시작하는 MCP_NO 앞부분만 말하면 정확히 같은 값이 아니라 MCP_NO에 해당 token이 포함된 제품을 찾도록 helper에 맡겨.
한 질문에 여러 제품이 콤마로 구분되어 들어오면 콤마 기준으로 제품 token 묶음을 나누고, 제품 묶음 내부는 모두 만족해야 하며 여러 제품 묶음은 OR 조건으로 매칭해.
RG, CP 같은 짧은 영문 token은 TECH 또는 FAMILY를 우선 보고, 16G 같은 용량 token은 DEN 또는 DENSITY, DDR4 같은 mode token은 MODE, FCBGA 같은 package token은 PKG_TYPE1 또는 PKG1, SDP/DDP 같은 package token은 PKG_TYPE2 또는 PKG2, 96 같은 숫자 token은 LEAD를 우선 매칭해.

다중 helper 형식 확인용 더미 function case를 등록해줘.
section은 pandas_function_cases이고 key는 sample_passthrough_demo야.
display_name은 다중 helper 형식 확인 더미야.
function_name은 sample_passthrough_helper야.
이 case는 실제 운영 분석 로직에는 사용하지 않고, 여러 pandas_function_cases가 의도 분석 결과와 pandas prompt에 어떻게 전달되는지 확인할 때만 선택해.
운영 질문에서는 사용자가 명시적으로 테스트나 형식 확인을 요청한 경우에만 선택해.
