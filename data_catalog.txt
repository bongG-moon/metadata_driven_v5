Goodocs 문서 ID는 1212121212121212121212 이야
목표2 문서에는 DATE, Mode, DEN, TECH, PKG1, PKG2, LEAD, ORG, MCP NO, INPUT 계획, OUT 계획 컬럼이 있어.

filter_mappings는 DATE -> DATE, MODE -> Mode, DEN -> DEN, TECH -> TECH, ORG -> ORG, PKG_TYPE1 -> PKG1, PKG_TYPE2 -> PKG2, LEAD -> LEAD, MCP_NO -> MCP NO로 연결해줘.

'INPUT 계획'은 투입계획이라고도 부르고 'OUT 계획'은 TARGET,생산목표라고도 불러.

LOT Status

text
LOT 정보조회 데이터는 lot_status로 등록해줘.
화면에 보일 이름은 LOT Status이면 돼.
현재 재공에서 작업중 Lot 수량, 작업대기 Lot 수량, Hold Lot 수량, Lot별 TAT를 확인하는 Oracle 데이터야.
이 데이터는 필수 조회 조건이 없어. (LOAD_DT와 BASE_DT는 단순 확인용 값)
수량/지표로는 PROD_QTY, WF_QTY, IN_TAT, CUM_TAT를 사용해.
특정 LOT들을 물어볼때는 LOT_ID컬럼에 IN조건을 걸면 돼
Lot 건수는 LOT_ID를 count해야하고,
DIE수량이나 UNIT수량을 물어볼때는 PROD_QTY값의 합을 구해야해.
그리고 Wafer수량, 웨이퍼수량을 물어볼때 사용하는게 WF_QTY컬럼이야.

IN_TAT는 현재 공정 유입 이후 TAT이고 CUM_TAT는 누적 TAT야.
HOLD여부는 HOLD_STAT컬럼이 'OnHold'값을 가지는 경우를 말해(OnHold, NotOnHold)

source는 oracle이고 db_key는 PNT_RPT야.

query_template:
/*Current Wip Status*/
SELECT
ERM_ID,OPER,OPER_NAME,FAB,OWNER,GRADE,DEVICE,LOT_ID,SUB_LOT_ID,PROD_QTY,WF_QTY,IN_TAT,CUM_TAT,EQP_ID,FLOW_ID,OPER_IN_TM,FAC_IN_TIME,HOLD_STAT,HOLD_REASON,FAMILY,MODE,DENSITY,TECH,ORG,PKG1,PKG2,PKG3,LEAD,MCP_NO,THK_CD,LOT_STAT,LOT_GRP,PKG_SIZE,HOT_LOT,HOT_LEVEL,PKG_COMPOSIT,DURABLE_ID,DURABLE_TYP,SUB_QTY,TSV_DIE_TYPE,EVENT_DESC,MOVE_IN_TM,PAD_ABNORMAL,SWR_REQ_NO,INSP_TARGET
FROM WIP_STATE
WHERE 1=1

filter_mappings는 OPER_NAME -> OPER_NAME, OPER_NUM -> OPER, MODE -> MODE, DEN -> DENSITY, TECH -> TECH, ORG -> ORG, PKG_TYPE1 -> PKG1, PKG_TYPE2 -> PKG2, LEAD -> LEAD, MCP_NO -> MCP_NO, TSV_DIE_TYP -> TSV_DIE_TYPE, EQP_ID -> EQP_ID, LOT_ID -> LOT_ID로 연결해줘.
